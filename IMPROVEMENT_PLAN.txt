Executive summary
-----------------
The app currently builds and its unit tests pass, which is a good base. The biggest improvement need is not a single bug. It is alignment: the repository mixes a small localhost-first Next.js/SQLite publishing app with a large set of aspirational platform, SaaS, cloud, SDK, and experimental API artifacts.

The best path is to stabilize the real working product first, then promote or retire the experimental platform pieces deliberately. Priority should be: production packaging, dependency security, source/type coverage, database consistency, API contract compatibility, and documentation cleanup.
